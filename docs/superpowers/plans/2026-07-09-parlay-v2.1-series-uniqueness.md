# Parlay v2.1 — Per-Series Uniqueness (Kalshi Combo Category Rule) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enforce Kalshi's combo rule — max 1 leg per market series (event ticker) per slip — in the v2 selection engine, version the ledger as `v2.1-combo`, and relock the 3 remaining QF slips as purchasable combos.

**Architecture:** Selection-time constraint in `selectSlipV2` (always on — old records are never re-selected); ledger versioned via new `engineVersion: "v2.1-combo"` + stored `maxLegsPerSeries: 1` so the inspector validates each slip against its OWN stored rule; v2-combo records stay untouched (never-delete) and grade under their own floors; UI badges v2-combo as legacy.

**Empirical ground truth (Kalshi collections API, fetched 2026-07-09 19:50Z):**
`GET /trade-api/v2/multivariate_event_collections?status=open` → collection `KXMVESPORTSMULTIGAMEEXTENDED-R`:
- Every modeled WC event (KXWCGAME, KXWCSPREAD, KXWCTOTAL, KXWCBTTS, KXWC1H, KXWC1HSPREAD, KXWC1HTOTAL, KXWC1HBTTS, KXWCADVANCE) has `size_max: 1` → **max 1 leg per series per match**.
- 1H series are separate events from FT series → TOTAL + 1HTOTAL in one slip is LEGAL.
- `yes_only: true` on KXWCGAME and KXWC1H (matches existing YES_ONLY_SERIES); KXWCGOAL also yes-only (unmodeled, irrelevant).
- Collection `size_min: 2` (min 2 legs) — already satisfied by existing floors.
- All 4 locked v2-combo slips violate size_max=1 (FRA-MAR 2×SPREAD+2×TOTAL, NOR-ENG 2×TOTAL, ESP-BEL 3×TOTAL, ARG-SUI 2×1HSPREAD) → not purchasable as one ticket.
